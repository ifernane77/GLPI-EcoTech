## Justification

Les tickets sont adaptés à une petite entreprise comme EcoTech.

Ils couvrent plusieurs situations réalistes :

- problème matériel ;
- problème réseau ;
- problème applicatif ;
- demande d’accès ;
- gestion documentaire ;
- sauvegarde ;
- gestion du parc.

## Lien avec GLPI

Ces tickets seront saisis plus tard dans GLPI afin de montrer :

- la création d’un ticket ;
- la qualification de la demande ;
- l’affectation ;
- le suivi ;
- la résolution ;
- la clôture ;
- l’association avec un équipement ou un service.

## Preuves à produire plus tard

Lorsque GLPI sera installé, il faudra garder :

- une capture de la liste des tickets ;
- une capture d’un ticket incident ;
- une capture d’un ticket demande ;
- une capture d’un ticket clôturé ;
- une capture d’un équipement associé à un ticket ;
- une capture d’une procédure liée à un ticket.
