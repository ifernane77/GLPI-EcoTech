# Cycle de vie d’un ticket — GLPI EcoTech

## Objectif

Ce document explique simplement les étapes de vie d’un ticket dans GLPI EcoTech.

Il sert de base pour comprendre le suivi d’un incident ou d’une demande.

---

## Cycle prévu

| Étape | Description |
|---:|---|
| 1 | L’utilisateur déclare un incident ou une demande |
| 2 | Le ticket est qualifié |
| 3 | Le ticket est classé dans une catégorie |
| 4 | Le ticket est affecté au support |
| 5 | Le ticket est traité |
| 6 | La solution est documentée |
| 7 | Le ticket est résolu |
| 8 | Le ticket est clôturé |

---

    ↓
