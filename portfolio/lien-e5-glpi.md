# Lien avec l’épreuve E5 — GLPI EcoTech

## Objectif

Ce document explique le lien entre le projet GLPI EcoTech et l’épreuve E5 du BTS SIO.

GLPI EcoTech est surtout un projet de support informatique.

Il me permet de préparer des preuves liées à la gestion du parc, aux tickets et aux demandes utilisateurs.

---

## Compétence : gérer le patrimoine informatique

GLPI permet de référencer les équipements utilisés par EcoTech.

Exemples d’équipements prévus :

- PC fixes ;
- PC portables ;
- téléphones professionnels ;
- imprimante / scanner ;
- Livebox ;
- serveur GLPI de test.

Preuves possibles :

- inventaire prévisionnel ;
- schéma SI ;
- liste des équipements ;
- captures GLPI plus tard.

---

## Compétence : répondre aux incidents et demandes

GLPI sert à créer et suivre des tickets.

Exemples de tickets prévus :

- PC administratif lent ;
- imprimante indisponible ;
- accès Google Drive ;
- connexion Internet instable ;
- téléphone professionnel abîmé ;
- devis introuvable.

Preuves possibles :

- tickets prévisionnels ;
- catégories GLPI ;
- flux de support ;
- captures de tickets plus tard.

---

## Compétence : mettre à disposition un service informatique

GLPI peut être présenté comme un service de support interne.

Il permettra aux utilisateurs de déclarer un incident ou une demande.

Preuves possibles :

- procédure d’installation ;
- procédure d’accès ;
- préparation des utilisateurs ;
- profils et droits ;
- captures de GLPI installé plus tard.

---

## Compétence : travailler en mode projet

Le projet est construit progressivement avec :

- un dépôt GitHub ;
- un journal de bord ;
- des fichiers Markdown ;
- des commits ;
- des preuves datées.

Preuves possibles :

- journal de bord ;
- README ;
- commits Git ;
- captures GitHub.
