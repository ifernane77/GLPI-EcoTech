# Préparation du paramétrage utilisateurs — GLPI EcoTech

## Objectif

Ce document explique ce qui est préparé avant de créer réellement les utilisateurs dans GLPI.

À cette étape, je prépare l’organisation.

La création réelle dans GLPI sera faite quand l’outil sera installé et accessible.

---

## Ce qui est préparé cette semaine

Cette semaine prépare :

- l’entité principale EcoTech ;
- les groupes utilisateurs ;
- les comptes prévus ;
- les profils GLPI ;
- les droits simples ;
- les règles de sécurité de base.

---

## Lien avec les leçons de la semaine

Les leçons de la semaine portent sur :

- Windows Server 2019 ;
- Active Directory ;
- utilisateurs ;
- groupes ;
- ordinateurs.

Même si GLPI n’est pas Active Directory, la logique est proche :

- on crée des utilisateurs ;
- on les range dans des groupes ;
- on leur attribue des droits ;
- on limite les accès selon leur rôle.

---

## Ce qui sera fait plus tard dans GLPI

Quand GLPI sera prêt, je pourrai :

1. créer l’entité EcoTech ;
2. créer les groupes ;
3. créer les utilisateurs ;
4. attribuer les profils ;
5. vérifier les droits ;
6. tester la création d’un ticket avec un compte demandeur ;
7. tester le traitement avec un compte technicien.

---

## Ce que je ne fais pas encore

Je ne crée pas encore de comptes réels dans GLPI si l’installation n’est pas terminée.

Je ne note aucun mot de passe dans GitHub.

Je ne mets pas de données personnelles réelles.

---

## Remarque personnelle

Je préfère préparer cette partie dans la documentation avant de la faire dans GLPI.

Cela me permettra d’éviter de créer des comptes au hasard et de mieux expliquer mon organisation au jury.
